## FORMAT SPECIFICATION SUMMARY

### YAML Frontmatter Schema (Mandatory)

```yaml
---
name: "[Agent_Name_Snake_Case]"
description: "[1–2 sentence description with trigger conditions and strategic function]"
trigger_conditions:
  - "[Condition 1]"
  - "[Condition 2]"
  - "[Condition 3]"
activation_mode: "AUTO / HITL / AUTO + HITL / MANUAL"
---
```

### Section Structure (Mandatory — 10 Sections)

| # | Section | Tone | Content Depth |
|---|---|---|---|
| I | Conceptual Foundation (WHY) | Strategic/Visionary | 3 subsections |
| II | Structural Breakdown (WHAT) | Policy/Technical | 3 subsections + diagrams |
| III | Operational Guidance (HOW) | Process/Technical | 4 subsections |
| IV | Structured Output Templates | Technical/Standard | 2–4 YAML templates |
| V | Reference Bank | Policy/Grounded | 3 subsections |
| VI | Quality Checklist | Compliance/Standard | 10–20 items table |
| VII | Integration Points | Technical/Architectural | 3 subsections |
| VIII | Governance & Enforcement | Policy/Compliance | 3 subsections (M, SP, HC) |
| IX | Success Metrics (KPIs) | Strategic/Measurable | 8–12 items table |
| X | Version Control | Administrative | Version history table |

### Governance Taxonomy (Mandatory)

- **HITL:** HC-1 to HC-110 (sequential, no gaps)
- **Mandatory:** M-1 to M-180 (sequential, no gaps)
- **Strict Prohibitions:** SP-1 to SP-180 (sequential, no gaps)
- **Quality:** Q-1 to Q-180 (sequential, no gaps)

### Tone Calibration

| Section | Tone | Language |
|---|---|---|
| I (Conceptual) | Strategic, visionary | "Transforms...", "Delivers...", "Enables..." |
| II (Structural) | Policy, technical | "MUST maintain...", "Auto-generates...", "Calculates..." |
| III (Operational) | Process, technical | "Step 1 —", "Auto-triggers when...", "Requires..." |
| IV (Templates) | Standard, precise | YAML with explicit typing |
| V (Reference) | Grounded, compliant | Indonesian regulations, GCG references |
| VI (Quality) | Compliance, rigorous | Checklist format, verification methods |
| VII (Integration) | Architectural, precise | Agent codes, data flows |
| VIII (Governance) | Policy, absolute | "MUST", "MUST NEVER", zero ambiguity |
| IX (KPIs) | Measurable, accountable | Numbers, frequencies, owners |
| X (Version) | Administrative | Standard version control table |

### Bilingual Convention

- **Employee-facing outputs:** Bilingual Indonesian + English (ID + EN)
- **Board/Executive outputs:** English primary, Indonesian summary
- **Technical/Operational outputs:** English primary
- **Regulatory citations:** Indonesian original + English translation

### File Naming Convention

```
SKILL_[Agent_Name_Snake_Case].md

Examples:
- SKILL_HR_Master_Orchestrator.md
- SKILL_Ethical_Guardrail_Sentinel.md
- SKILL_Predictive_Workforce_Simulator_Scenario_Planning_Engine.md
```
