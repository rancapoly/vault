# Crisis / Issue Briefing Deck
## Polytron Board Deck Reference

---

## TUJUAN DECK

Crisis/Issue Briefing adalah komunikasi darurat kepada Board ketika terjadi insiden
atau isu yang berpotensi berdampak signifikan terhadap bisnis, reputasi, atau regulasi.

Prinsip utama: **Clarity over elegance. Speed over perfection.**

Board membutuhkan 4 hal dalam urutan ini:
1. **Apa yang terjadi?** (faktual, tanpa spin)
2. **Seberapa besar dampaknya?** (quantified, worst-case included)
3. **Apa yang sudah dan akan kita lakukan?** (concrete actions + owners)
4. **Apa yang dibutuhkan dari Board?** (keputusan/dukungan spesifik)

---

## KLASIFIKASI KRISIS

Tentukan level krisis untuk menentukan urgensi dan depth deck:

| Level | Kriteria | Deck Turnaround |
|---|---|---|
| **CRITICAL** | Dampak revenue > 5%, reputasi publik, regulasi/hukum | 4–6 jam |
| **HIGH** | Dampak operasi > 48 jam, potential media exposure | 24 jam |
| **MEDIUM** | Dampak terlokalisir, risiko eskalasi moderat | 48–72 jam |

## STRUKTUR SLIDE (5–8 Slide)

### Slide 1: Cover — Crisis Identifier

```
Background: CHARCOAL (#2C2C2C) — bukan merah, agar terkesan serius bukan panik
Konten:
- Logo Polytron (putih, kiri atas)
- "ISSUE BRIEFING" — label merah kecil (tag), di atas judul
- "[NAMA INSIDEN / ISU]" — judul besar, putih, bold
- Level badge: [CRITICAL / HIGH / MEDIUM] — rectangle berwarna
- "Dilaporkan kepada: Top Management & Board of Directors"
- "Tanggal: [DD/MM/YYYY] | Jam: [HH:MM] WIB"
- "Penyusun: [Jabatan owner]"
- "STRICTLY CONFIDENTIAL" — kecil di bawah
```

---

### Slide 2: Situation at a Glance (1 Halaman — Wajib Berdiri Sendiri)

```
Header: "Situasi: [Ringkasan insiden dalam 1 kalimat tegas]"
Format: 4-box grid (2×2)

BOX 1 — APA YANG TERJADI (background merah gelap)
  Timeline ringkas: [Tanggal/jam pertama terdeteksi]
  Deskripsi singkat: [2-3 kalimat faktual tanpa asumsi]

BOX 2 — DAMPAK SAAT INI (background amber)
  - Finansial: Rp [X] potensi kerugian / revenue at risk
  - Operasional: [sistem/lini/proses yang terdampak]
  - Reputasi: [level exposure publik/media]
  - Regulasi: [potensi sanksi/pelanggaran]

BOX 3 — YANG SUDAH DILAKUKAN (background charcoal)
  - [Tindakan 1] — [jam/tanggal] — [Owner]
  - [Tindakan 2] — [jam/tanggal] — [Owner]
  - [Tindakan 3] — [jam/tanggal] — [Owner]

BOX 4 — STATUS SAAT INI (background sesuai level)
  Status: [CONTAINED / ACTIVE / ESCALATING]
  [1-2 kalimat status terkini]
  Next update: [waktu briefing berikutnya]
```

---

### Slide 3: Timeline Kejadian

```
Header: "Timeline: Bagaimana insiden ini berkembang"
Format: Horizontal timeline (kiri ke kanan)

[T-0] [T+1h] [T+3h] [T+6h] [T+12h] [SEKARANG] [T+X — Projected]

Setiap titik:
  - Waktu (bold)
  - Deskripsi kejadian (12pt)
  - Status: DETECTED / ESCALATED / CONTAINED / RESOLVED
  - Ikon status (lingkaran merah/amber/hijau)

Proyeksi ke depan (garis putus-putus):
  - Expected containment: [waktu]
  - Expected resolution: [waktu]
```

---

### Slide 4: Root Cause & Contributing Factors

```
Header: "Root Cause: [Penyebab utama] — [X] faktor berkontribusi"
Format: Fishbone / Why-Why analysis (sederhana, 2 level)

Primary Cause: [...]
  ├── Contributing Factor 1: [...]
  ├── Contributing Factor 2: [...]
  └── Contributing Factor 3: [...]

Catatan penting:
  "Root cause analysis ini bersifat PRELIMINARY.
   Investigasi penuh akan selesai pada [tanggal]."

Jika root cause belum diketahui:
  Ganti slide ini dengan "Open Questions" — apa yang masih diselidiki
```

---

### Slide 5: Impact Assessment

```
Header: "Dampak: [Kuantifikasi dampak bisnis secara total]"
Format: 3-kolom tabel

Kolom 1 — DAMPAK TERKONFIRMASI (data aktual):
  - Financial: Rp [X] kerugian/pendapatan hilang
  - Operasional: [X] jam/hari downtime
  - Customer: [X] pelanggan/dealer terdampak

Kolom 2 — RISIKO JIKA TIDAK DITANGANI (worst-case):
  - Financial: Rp [X] potensi eksposur
  - Reputasi: [tingkat exposure media/publik]
  - Regulasi: [potensi sanksi]

Kolom 3 — YANG SUDAH TERMITIGASI:
  - [Mitigasi 1]: Rp [X] dilindungi
  - [Mitigasi 2]: [proses dilindungi]

Bawah tabel: "Net exposure saat ini: Rp [X] — [naik/turun] dari estimasi awal"
```

---

### Slide 6: Action Plan (Wajib Ada)

```
Header: "Action Plan: [X] tindakan aktif dengan owner dan deadline jelas"
Format: Tabel 5 kolom

| # | Tindakan | Owner | Deadline | Status |
|---|---|---|---|---|
| 1 | [Immediate containment action] | [Jabatan] | [DD/MM HH:MM] | 🔴 In Progress |
| 2 | [Communication action] | [Jabatan] | [DD/MM] | 🟡 Pending |
| 3 | [Customer/dealer notification] | [Jabatan] | [DD/MM] | 🟢 Done |
| 4 | [Regulatory notification] | [Jabatan] | [DD/MM] | 🟡 Pending |
| 5 | [Root cause fix / prevention] | [Jabatan] | [DD/MM] | 🔴 In Progress |

Status color: Merah = In Progress (belum selesai), Amber = Pending, Hijau = Done
```

---

### Slide 7: Decisions Needed from Board

```
Header: "Kami memerlukan arahan Board pada [X] hal berikut"
Background: Deep charcoal (#1A1A1A) — pembeda visual penting

Format: Numbered list dengan impact box

1. [KEPUTUSAN YANG DIBUTUHKAN]
   Opsi A: [...] — Implikasi: [...]
   Opsi B: [...] — Implikasi: [...]
   Rekomendasi Management: [Opsi X karena ...]

2. [OTORISASI ANGGARAN jika diperlukan]
   Estimasi kebutuhan: Rp [X]
   Untuk: [keperluan spesifik]

3. [KOMUNIKASI EKSTERNAL — jika ada]
   Apakah Board menyetujui [pernyataan/press release/notifikasi regulasi]?
```

---

### Slide 8: Next Update & Closing Commitment

```
Header: "Kami berkomitmen untuk memberikan update selanjutnya pada [waktu/tanggal]"
Format: 3 commitment boxes

BOX 1 — NEXT BRIEFING:
  [Tanggal] [Waktu] — format [meeting/laporan tertulis]
  Agenda: [apa yang akan dilaporkan]

BOX 2 — RESOLUTION TARGET:
  Containment: [tanggal target]
  Full Resolution: [tanggal target]
  Prevention in place: [tanggal target]

BOX 3 — ESCALATION TRIGGER:
  Kami akan segera menginformasikan Board jika:
  - [kondisi eskalasi 1]
  - [kondisi eskalasi 2]
```

## CATATAN PPTX KHUSUS CRISIS DECK

- **Warna dominan**: Charcoal (#2C2C2C) dan merah gelap — bukan merah cerah (agar serius)
- **JANGAN** membuat slide yang terlalu "polished" — terlihat tidak tulus saat krisis
- **Action table**: Wajib ada kolom STATUS dengan traffic light yang jelas
- **Timeline**: Gunakan shape horizontal + text box, bukan chart (lebih fleksibel)
- **Slide 2 (Situation at a Glance)**: Slide ini HARUS bisa berdiri sendiri — Board kadang hanya membaca ini
- **Tone**: Faktual, akuntabel, forward-looking — BUKAN defensive atau menyalahkan pihak lain
