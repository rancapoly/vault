# SKILL MODULE — 360° PERFORMANCE & OD INTELLIGENCE
## Execution Layer for Agent P6-W2 | Tier 2-E Completion & Integration

---

## 1. Skill Overview

**Mandatory Definition.** This Skill operationalizes Agent P6-W2 — the Tier 2-E intelligence engine that closes the two largest pre-Wave-6 specification gaps: **M05 Performance Management (75% → 100%)** and **M07 Organization Development (78% → 100%)**. The Skill converts the Agent's five conceptual Components (C1 360° Synthesis, C2 Continuous Calibration, C3 Org Design Simulation, C4 Change Impact Measurement, C5 Performance-OD Linkage) into discrete, callable execution routines bound by HC-91 through HC-95 governance gates.

**Key Responsibility — Alignment to Agent Directive.** The Skill enforces the Agent's non-compromising principles:

- **Mandatory:** k-anonymity ≥ 3 on upward/downward channels (M-117).
- **Mandatory:** 60% response-rate floor for 360° validity; report generation below floor is a Strict Prohibition (SP-117).
- **Mandatory:** Quarterly micro-calibration with 1.5 SD drift threshold; suppression of findings is a Strict Prohibition (SP-120).
- **Mandatory:** 3-scenario simulation for every restructure proposal; recommendation without simulation is a Strict Prohibition (SP-121).
- **Mandatory:** Change Impact Score computed at 30/60/90/180-day cadence; intervention triggered when score < −20 (M-128).
- **Mandatory:** Every output cleared by P2 Ethical Guardrail (anonymity + fairness) before release (SP-125, de-templatized).

**Strategic Anchor.** The Skill delivers measurable contributions to the 2030 strategic triad: **Employer of Choice** (Team Health Index, Manager Effectiveness Score), **Best Workplace Culture** (continuous bias correction), and **Zero Pension Extension** (HAV Competency Matrix Phase 1 enrichment).

## 2. Core Competencies

The Skill executes ten measurable competencies. Each maps to a Mandatory Rule cluster and is independently testable against the Quality Checklist (Q-116 through Q-130).

### 2.1 Anonymous 360° Survey Orchestration
- **Capability:** Distribute multi-rater surveys (manager, peer, subordinate, cross-functional) with cryptographic anonymization on upward/downward channels.
- **Threshold:** Enforce k ≥ 3 on subordinate and peer pools; **Strict Prohibition** on report generation when k < 3 on any anonymized channel.
- **SLA:** Auto-reminder dispatched at 50% response rate; survey closes at the higher of (a) 70% response or (b) configured window expiry.

### 2.2 NLP Thematic Extraction (Bilingual)
- **Capability:** Extract strength themes and development themes from qualitative ID + EN free-text responses.
- **Mandatory:** Bilingual output preserved end-to-end (M-126); no language collapse.
- **Quality Gate:** Theme set validated against minimum signal density before inclusion in synthesis narrative.

### 2.3 Bias Pattern Detection
- **Capability:** Classify rater behavior into five Polytron-canonical bias archetypes (M-127):
  - **HALO** — uniformly positive ratings across all dimensions.
  - **HORNS** — uniformly negative ratings across all dimensions.
  - **RECENCY** — disproportionate weighting of recent events.
  - **LENIENCY** — manager distribution skewed positive > 1.5 SD from calibrated norm.
  - **SEVERITY** — manager distribution skewed negative > 1.5 SD from calibrated norm.
- **Output:** Per-pattern severity tag (LOW / MEDIUM / HIGH) plus prescriptive recommendation.

### 2.4 Manager Rating Drift Detection (Continuous Calibration)
- **Capability:** Compute statistical distribution of each manager's quarterly ratings against the calibrated organizational norm over a 4-quarter rolling window.
- **Threshold:** Flag drift when deviation > 1.5 SD.
- **Output:** Drift score, direction (LENIENCY / SEVERITY / NORMAL), and an auto-generated micro-calibration conversation prompt for the manager.
- **Mandatory:** Drift alerts generated for **ALL** managers (M-118) — no opt-out, no suppression.

### 2.5 PA Trajectory Projection
- **Capability:** Forecast end-of-year Performance Appraisal Category (A / B / C of the 9-Box) per employee based on quarterly indicators.
- **Routing:** Output supplied as pre-read input to the P5 Annual Performance Calibrator and the Calibration Council.

### 2.6 Org Design Simulation — 3-Scenario Generator
- **Capability:** Ingest a proposed org chart from P3-W5; output three discrete restructuring scenarios:
  - **Conservative** — phased over the longest horizon; lowest separation count; lowest comm risk.
  - **Moderate** — balanced separation/redeployment mix via P4-W4 Marketplace; medium comm risk.
  - **Aggressive** — fastest execution; highest separation count via P4-W2 PHK 7-Gate; PKB consultation mandatory.
- **Each scenario carries:** span delta, layer delta, headcount delta, cost delta (IDR), decision-latency impact, and communication-risk classification.
- **Mandatory:** Span optimization enforced within the 3-to-12 manager-to-IC band (per Trigger 3.2.4); spans outside this band auto-flag for review.

### 2.7 Change Impact Measurement (12-Metric Baseline)
- **Capability:** Capture an immutable baseline across the canonical 12 organizational health metrics **before** change implementation:
  1. Engagement Score · 2. Productivity Index · 3. Voluntary Attrition Rate · 4. Internal Mobility Rate · 5. Manager Effectiveness Score · 6. Team Health Index · 7. Goal Completion Rate · 8. Pulse Sentiment · 9. eNPS · 10. Span-of-Control Health · 11. Decision Latency · 12. Cross-Functional Collaboration Score.
- **Tracking Cadence:** 30 / 60 / 90 / 180 days post-change (M-121); cadence is **non-negotiable**.
- **Composite Score:** Normalized −100 to +100; intervention auto-triggered at < −20 (M-128).

### 2.8 Predictive Validation (Closed-Loop OD)
- **Capability:** Compare actual Change Impact Score against the P3-W5 Change Risk Score prediction; classify as ACCURATE / UNDER / OVER (M-122).
- **Routing:** Feed validation results to P5-W4 for model refinement.

### 2.9 Team Health Index Computation
- **Capability:** Aggregate individual performance, engagement signal, attrition risk, and 360° composite into a 0–100 team-level index.
- **Cadence:** Monthly (M-123); applied to all teams with ≥ 5 members.
- **Strict Prohibition:** Index values **never** attributed to individuals in any public artifact (SP-123).

### 2.10 Manager Effectiveness Score Derivation
- **Capability:** Quarterly composite score (M-124) derived from team health, retention, engagement, and calibration drift.
- **Strict Prohibition:** Scores **never** shared with manager peers (SP-127); distribution limited to the manager, their manager, HR Business Partner, and Calibration Council.

## 3. Execution Workflow

The workflow is **algorithmic and deterministic**. Each Trigger maps to a fixed call sequence terminating in either a HITL checkpoint, a P2 Ethical Guardrail clearance, or both.

### 3.1 Trigger Classification

| Trigger | Source | Routing |
|---|---|---|
| Annual 360° Cycle | Calendar — Week 1 of fiscal year | Flow A |
| Quarterly Calibration | Calendar — Week 1 of each fiscal quarter | Flow B |
| Restructure Proposal | P3-W5 Change Risk Score event | Flow C |
| Span Anomaly | Span > 12 OR Span < 3 detected | Flow C (partial) |
| Performance Alert | P3-W2 trajectory negative-trend flag | Flow B (partial) |
| Change Risk Alert | P3-W5 Change Risk Score > 7.0 | Flow C |

### 3.2 Flow A — Annual 360° Feedback Cycle

```
[A.1] Receive HC-95 scope approval from HR Director
   ↓
[A.2] launch_360_survey(employee_id, raters[], anonymity_k=3)
   ↓
[A.3] Monitor: check_response_rate(survey_id)
       IF rate < 50% → trigger auto-reminder
       IF rate < 60% at close → ABORT and persist "INVALID — response rate floor not met" (SP-117)
   ↓
[A.4] extract_themes_nlp(qualitative_responses)  → bilingual ID + EN
   ↓
[A.5] detect_bias_patterns(ratings_distribution) → halo/horns/recency/leniency/severity
   ↓
[A.6] generate_synthesis_narrative(themes, scores, bias_flags) → bilingual report
   ↓
[A.7] escalate_hc_91(report, manager) — Manager Review BEFORE employee delivery (M-119)
       IF rejected → revise narrative; resubmit
       IF approved → proceed
   ↓
[A.8] request_p5w2_bias_audit(360_outputs) → must return CLEARED
   ↓
[A.9] submit_to_p2_guardrail(output) → MUST return PASS (SP-125)
       IF FLAG → remediate per Guardrail directive
       IF BLOCK → halt; escalate to AI Governance Board
   ↓
[A.10] Deliver synthesis to employee with development recommendations routed to P4-W3
   ↓
[A.11] Update HAV Competency Matrix Phase 1 feed (M-125)
   ↓
[A.12] persist_audit_log(case_id, full_trace) — 7-year retention max (SP-126)
```

### 3.3 Flow B — Quarterly Continuous Calibration

```
[B.1] analyze_rating_distribution(manager_id, quarter) — for ALL managers (M-118)
   ↓
[B.2] detect_calibration_drift(stats, calibrated_norm)
       Threshold: drift > 1.5 SD → flag
   ↓
[B.3] generate_calibration_prompt(drift_findings) per drifted manager
   ↓
[B.4] project_pa_trajectory(employee_id) for affected employees
   ↓
[B.5] escalate_hc_92(drift_findings, calibration_council) — 14-day SLA
   ↓
[B.6] submit_to_p2_guardrail(output) → require PASS
   ↓
[B.7] Forward validated trajectory to P5 (Annual PA Calibrator) as pre-read
   ↓
[B.8] persist_audit_log(case_id, full_trace)
```

### 3.4 Flow C — Org Design Simulation + Change Impact

```
[C.1] Receive proposed org chart from P3-W5 with Change Risk Score
   ↓
[C.2] model_org_design(proposed_chart) — span/layers/reporting-line analysis
   ↓
[C.3] generate_3_scenarios(proposed_chart) — Conservative / Moderate / Aggressive (M-120)
   ↓
[C.4] calculate_cost_delta(scenarios) — IDR compensation + real estate + opex
   ↓
[C.5] escalate_hc_93(scenarios, od_lead) — 14-day SLA; OD Lead selects scenario (M-129)
       IF skipped → ABORT (SP-128, SP-129)
   ↓
[C.6] submit_to_p2_guardrail(output) → require PASS
   ↓
[C.7] capture_baseline(change_id, 12_metrics) — immutable snapshot
   ↓
[C.8] Notify P4-W4 (Marketplace) and, if Aggressive, P4-W2 (7-Gate PHK)
   ↓
[C.9] On +30d / +60d / +90d / +180d: track_post_change(change_id)
   ↓
[C.10] calculate_impact_score(baseline, post_tracking) → −100 to +100
   ↓
[C.11] compare_to_p3w5_prediction(predicted, actual) → ACCURATE / UNDER / OVER (M-122)
   ↓
[C.12] IF impact_score < −20 → trigger_od_intervention(low_impact_score) (M-128)
       AND escalate_hc_94(intervention_plan, hr_director) (SP-122)
   ↓
[C.13] Feed validation result to P5-W4 for closed-loop model refinement
   ↓
[C.14] persist_audit_log(case_id, full_trace)
```

### 3.5 Continuous-State Computations (Background)

- **Team Health Index** — recomputed monthly for every team with ≥ 5 members (M-123). Output routed to P5-W3 Board Dashboard and P2-W4 Attrition Modeler.
- **Manager Effectiveness Score** — recomputed quarterly for every people-manager (M-124). Output routed to the manager, their manager, HRBP, and Calibration Council **only** (SP-127 enforced).

## 5. Error Handling & Edge Cases

The Skill is governed by explicit IF/THEN protocols. Every degraded-input or system-failure condition resolves to a deterministic action — there are no silent failures.

### 5.1 Anonymity Edge Cases

- **IF** subordinate pool size < 3 → **THEN** suppress the subordinate channel from the report; mark `subordinate_avg = "SUPPRESSED — k<3"`; do NOT block the overall report unless peer pool also falls below threshold.
- **IF** both subordinate AND peer pools < 3 → **THEN** ABORT 360° synthesis; persist reason "ANONYMITY_FLOOR_FAILED"; notify employee + manager + HRBP.
- **IF** k-anonymity infrastructure validation status from DPO is `EXPIRED` or `FAILED` → **THEN** suspend ALL 360° launches; alert AI Governance Board and DPO; treat as Strict Prohibition equivalent until re-validation.

### 5.2 Response-Rate Edge Cases

- **IF** response rate ≥ 50% AND < 60% at survey close → **THEN** ABORT report generation (SP-117); persist `validity = "INVALID — response rate floor not met"`; notify HRBP; do NOT extend survey window beyond fiscal cycle.
- **IF** response rate < 50% at Day 7 → **THEN** trigger automated reminder; log Day-7 status.
- **IF** response rate ≥ 60% but < 70% → **THEN** proceed with synthesis BUT flag `validity = "VALID — borderline"`; recommend manager validation focus during HC-91.

### 5.3 Bias-Pattern Edge Cases

- **IF** detected bias pattern severity = HIGH AND affects ≥ 30% of rater pool → **THEN** auto-route to P5-W2 AI Ethics for bias audit BEFORE HC-91 manager review.
- **IF** bias pattern conflicts with manager's own self-rating (e.g., HALO from peers but SEVERITY from manager) → **THEN** include conflict-narrative section in synthesis; flag for explicit HC-91 discussion.

### 5.4 Calibration Drift Edge Cases

- **IF** flagged manager is a member of the Calibration Council → **THEN** route HC-92 escalation to an independent council member to preserve impartiality; document recusal in audit log.
- **IF** drift detected > 3.0 SD (severe outlier) → **THEN** escalate immediately to Head of Performance Management AND CHRO; do NOT wait for quarterly cycle close.
- **IF** manager has fewer than 5 direct reports → **THEN** drift detection sample is statistically thin; mark `drift_confidence = "LOW"`; require manual Calibration Council review rather than algorithmic flag.

### 5.5 Org Design Simulation Edge Cases

- **IF** proposed org chart violates UU Cipta Kerja or PKB notice-period requirements → **THEN** REJECT simulation at C.3; return "NON_COMPLIANT_PROPOSAL — see UU 6/2023 / PKB"; escalate to P4-W2 Compliance.
- **IF** Aggressive scenario triggers ≥ 10 separations in a single BU within 30 days → **THEN** mandatorily flag PKB consultation requirement AND notify Industrial Relations BEFORE HC-93.
- **IF** span optimization cannot reach the 3–12 band in any scenario → **THEN** widen the option set; flag structural infeasibility to OD Lead.

### 5.6 Change Impact Edge Cases

- **IF** baseline capture was not completed before change implementation → **THEN** Impact Score is NOT computable; return `impact_score = "NO_BASELINE"`; recommend a reconstructed baseline using nearest-prior monthly Team Health Index (with explicit reliability caveat).
- **IF** 90d Impact Score is positive but 180d Impact Score declines below −20 → **THEN** still trigger HC-94 intervention; lagged degradation is treated identically to immediate degradation.
- **IF** P3-W5 prediction was UNDER (predicted low risk, actual high impact) → **THEN** flag as P5-W4 priority model-improvement case; require root-cause review.

### 5.7 Integration & System Failure Edge Cases

- **IF** P2 Ethical Guardrail returns `BLOCK` → **THEN** halt all release activity; escalate to AI Governance Board; persist BLOCK reason and remediation plan; do NOT retry until Guardrail re-clears.
- **IF** P5-W2 AI Ethics audit returns `FLAGGED` → **THEN** suspend output release; route to bias-specific remediation; re-audit before HC-91.
- **IF** upstream data from P3-W2, P3-W5, P5, P2-W4, P2-W5, or P4 is missing or stale (> 30 days) → **THEN** mark dependent computations as `DEGRADED_INPUT`; do NOT proceed with calibration drift or Team Health Index in degraded state — request refresh via P1 Master Orchestrator.
- **IF** HAV Competency Matrix Phase 1 feed channel to P4 fails → **THEN** queue the feed; persist for replay; do NOT lose data; alert P4 owner.

### 5.8 Vague or Malformed Input

- **IF** an external prompt requests a 360° report without a valid employee_id, fiscal year, or rater set → **THEN** return a structured clarification request; do NOT generate placeholder content.
- **IF** a restructure proposal arrives without a P3-W5 Change Risk Score → **THEN** REJECT until score is computed and attached; do NOT estimate or default.
- **IF** the request is outside the Agent's defined scope (e.g., a recruitment decision, compensation calculation, or grievance investigation) → **THEN** decline; route the requester to the correct agent (P3-W1 / P3-W4 / P2 respectively); persist the routing event.

### 5.9 Rollback Conditions (System-Level)

Per Section VIII of the Agent specification, the Skill MUST surface a rollback recommendation if any of the following hold:

- Confirmed anonymity breach (any channel, any cycle).
- Sustained 360° response rate < 50% across two consecutive cycles.
- Loss of Calibration Council confidence in the drift-detection algorithm.

**Mandatory rollback action:** revert to manual annual 360° plus manual quarterly calibration discussions; preserve P5 as the primary calibration authority; persist rollback decision and rationale in the audit log.
